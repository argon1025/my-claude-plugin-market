# pr-review Layer 1 체크리스트 — 공통 (언어 불문)

- 모든 스택에 적용. 스택별 체크리스트(typescript-react.md, java-spring.md, nestjs.md)와 병용
- Layer 2(프로젝트 문서)가 같은 주제를 다루면 Layer 2 기준으로 판정
- 전 항목 공통: 보고 조건 충족 AND 미보고 조건 미해당일 때만 보고
- lint 제외: 이 프로젝트의 lint 설정(eslint·checkstyle 등)이 error로 차단하는 항목은 보고 금지
- 등급 3단은 리뷰 프롬프트의 판정 질문을 따른다: Blocker(그대로 배포 시 하위호환 파괴·크래시·보안 노출 — 수정 후 배포) / Bug(잘못 동작하지만 배포 차단 아님) / 개선(결함 아님 — 설계·컨벤션·성능 제안)

## 결함

### 공개 계약 변경 미반영
- 보고 조건: 변경된 공개 시그니처·API 응답 필드·이벤트 payload·설정 키의 기존 소비처가 구계약 기준으로 남음 — 소비처를 Grep/`git show`로 확인한 뒤에만 판정
- 미보고 조건: 소비처 전부가 같은 PR에서 갱신됨, 소비처 0곳
- 등급 가이드: 배포 즉시 기존 소비처가 깨지면 Blocker, 특정 조건·입력에서만 깨지면 Bug

### 시크릿 하드코딩
- 보고 조건: API 키·토큰·비밀번호·서명키 리터럴이 코드·배포 산출물 노출 경로에 존재
- 미보고 조건: 공개 전제 값(public key, 테스트 전용 더미로 명시)
- 등급 가이드: Blocker

### 에러 삼킴
- 보고 조건: 빈 catch, catch 후 무의미 기본값 반환으로 실패가 침묵
- 미보고 조건: 무시 사유 주석 명시+무시가 계약(선택 기능 폴백)
- 등급 가이드: Bug

### 비동기 실패 미처리·race
- 보고 조건: 대기·실패 처리 없는 비동기 호출(floating promise 등), 획득한 리소스(구독·타이머·핸들) 해제 누락, 연속 요청 race로 뒤늦은 응답이 최신 상태를 덮어씀
- 미보고 조건: fire-and-forget이 의도이고 실패가 무해, 취소·해제 경로 존재
- 등급 가이드: Bug — race로 사용자 데이터가 잘못 저장·유실되는 경로가 확인되면 Blocker

## 설계

### 중첩 삼항
- 보고 조건: 삼항 내부 삼항(2단 이상) 또는 삼항·논리연산자 혼합으로 분기 추적이 어려운 식
- 미보고 조건: 1단 삼항, 관례적 조건부 렌더·표현식 1단
- 등급 가이드: 개선

### 레이어 타입 디커플링
- 보고 조건: 외부 시스템 타입(API 응답·DB 스키마)을 표현·도메인 레이어가 직접 import하거나 캐스팅으로 레이어 관통
- 미보고 조건: 경계 Mapper 존재, 소비 측 덕타입(필요 필드만 구조 선언) 수용
- 등급 가이드: 개선

### 순수 로직 인라인 매몰
- 보고 조건: 매핑·판정·변환 분기 로직이 콜백·핸들러 인라인에 매몰되어 단위 테스트 불가
- 미보고 조건: 1~2줄 자명한 분기
- 등급 가이드: 개선

### Guard Clause 부재
- 보고 조건: 2단 이상 중첩 조건이 early return으로 평탄화 가능한데 늦은 리턴 유지
- 미보고 조건: 중첩 1단, 평탄화 시 의미 손실
- 등급 가이드: 개선

### 얕은 pass-through 래퍼
- 보고 조건: 내부 옵션을 그대로 노출해 인터페이스가 구현만큼 복잡한 wrapper 신설
- 미보고 조건: 복잡성 은닉·기본값 주입 등 실질 가치 있는 래퍼
- 등급 가이드: 개선

### YAGNI 위반
- 보고 조건: 1개 호출부 전용 옵션·분기를 공용 모듈에 추가, 미소비 값 전량 계산·반환(over-provisioning)
- 미보고 조건: 같은 PR 내 복수 호출부가 실사용
- 등급 가이드: 개선
