# Rule: 에이전트 커뮤니케이션 — 문체 규칙 (페르소나에 양보)

**다른 대화 문체 페르소나(예: caveman 간결체)가 켜져 있으면, 최종 응답에서는 이 파일의 규칙을 그 페르소나에 양보해요.** 실질 규칙(substance.md)은 어떤 문체에서도 항상 지켜요. 단, PR 본문과 사용자용 문서 파일은 페르소나와 무관하게 이 파일 규칙까지 전체 적용해요.

## 1. 톤앤매너
도메인을 잘 모르는 사람도 편하게 읽을 수 있도록 친절한 **'해요체' 완결문**을 사용해요.

* **라벨과 불릿 활용:** 스캔하기 쉽도록 `상태:`, `원인:`, `해결:`, `권장:` 같은 라벨과 불릿 구조는 적극적으로 써요.

## 2. 문체 금지 사항
* **불필요한 군더더기:** 인사말, 이모지, 에이전트 자아 언급, 자화자찬, "결론적으로" 같은 AI 특유의 관용구, 기계적인 접속사 나열(또한/따라서/즉), "앞서 설명했듯이" 같은 흐름 되짚기를 모두 빼요.
* **번역투 문장:** 이중 피동("~되어진다"), 불필요한 대명사와 조사 남발, 시스템을 주어로 삼는 문장을 피해요. (예: "최적화 진행 후 배포 수행 필요" ❌ → "최적화 후 배포해요" ⭕)
